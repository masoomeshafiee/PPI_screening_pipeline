# AlphaFold Server

## Overview

This document describes how to use AlphaFold Server together with the pooled-PPI workflow.

For most users, AlphaFold Server is the recommended way to run pooled-PPI because it requires:

* No AlphaFold installation
* No HPC access
* No GPU resources
* No container setup

The pooled-PPI workflow generates AlphaFold-compatible JSON files that can be submitted directly to AlphaFold Server.

The workflow is:

```text
candidate_proteins.fasta
          ↓
Pool generation
          ↓
AlphaFold Server JSON generation
          ↓
Input validation
          ↓
Upload to AlphaFold Server
          ↓
Download prediction results
          ↓
Score extraction
          ↓
Size correction
          ↓
Final ranked interaction table
```

---

# When to Use AlphaFold Server

AlphaFold Server is recommended for:

* Small to medium screens
* Pilot studies
* Method development
* Users without HPC access
* Users who want the simplest workflow

Advantages:

* No installation required
* No model weights required
* No database downloads required
* No resource management
* No SLURM knowledge required

Limitations:

* Manual upload and download
* Daily job quota
* Less automation
* Not practical for very large-scale screens

For genome-scale screens, the HPC workflow is generally more suitable.

See:

```text
docs/alphafold3_hpc.md
```

# Step 1 — Generate AlphaFold Inputs

Run the pooled-PPI preparation stage:

Modify the config.yaml in the configs directory. 

Then in your terminal:

```bash
python src/workflow.py --config configs/config.yaml
```

with:

```yaml
workflow:
  stage: prepare
```

This produces:

```text
alphafold3_jsons/
```

containing one JSON file per pool.

Example:

```text
alphafold3_jsons/
├── pool_00001.json
├── pool_00002.json
├── pool_00003.json
└── ...
```

---

# Step 2 — Configure AlphaFold Server Output Format

In the configuration file:

```yaml
alphafold3:
  dialect: alphafoldserver
```

The JSON builder automatically generates the format expected by AlphaFold Server.

Example:

```yaml
alphafold3:
  dialect: alphafoldserver
  model_seeds: [1]
```

---

# Step 3 — Validate Inputs

Before submitting jobs, validate the generated JSON files:

```bash
python src/workflow.py --config configs/config.yaml
```

Validation checks:

* JSON structure
* Sequence presence
* Duplicate identifiers
* Pool size limits
* Chain mapping consistency
* AlphaFold Server compatibility

Output:

```text
af3_json_validation_summary.tsv
```

Any validation errors should be fixed before running AlphaFold.

---

# Step 4 — Upload Jobs to AlphaFold Server

Open:

```text
https://alphafoldserver.com
```

Upload one JSON file at a time.

Example:

```text
pool_00001.json
```

or
batch_001.json
...

Each pool corresponds to one AlphaFold prediction job.

You can also upload a the batch file created in the same folder as the input json files by the json builder. for instance: batch_001.json
This batch file has up to 100 pools/jobs inside. Alphafold server automatically detects all the jobs inside and load them as draft. Then you can run each draft. The maximum number of files to be drafted is 100 files. That is why the json builder in our pipeline generates batched of max 100 jobs. 

Using the batch json files can minimize the manual work compared to uploading individual jobs one by one. 

---

# Important Server Limits

AlphaFold Server currently imposes usage limits.

These limits may change over time.

At the time of development:

* Approximately 30 jobs per day per Google account
* Manual upload required
* Manual download required

For large projects, multiple days may be required to process all pools.

Always verify current limits directly on the AlphaFold Server website.

---

# Step 5 — Monitor Job Status

After submission, AlphaFold Server places jobs into a queue.

Typical states include:

```text
Drafts (queue)
Running
Completed
Failed
```

Prediction time depends on:

* Number of proteins
* Protein lengths
* Server load
* Queue length

  if a job failed, you can rerun it. 

---

# Step 6 — Download Results

After completion, download the full result package for each pool.

Do not extract only the structures.

The pooled-PPI workflow requires confidence information generated by AlphaFold.

---

# Recommended Organization

Create a prediction directory:

```text
alphafold3_predictions/
```

and place one folder per pool:

```text
alphafold3_predictions/
├── pool_00001/
├── pool_00002/
├── pool_00003/
└── ...
```

The final structure should contain all AlphaFold outputs for every pool.

---

# Required Files

The pooled-PPI workflow extracts interaction scores from AlphaFold confidence outputs.

Typical files include:

```text
*_summary_confidences.json
```

and other AlphaFold prediction outputs.

Do not delete any files from the downloaded result folders.

---

# Step 7 — Run pooled-PPI Processing

After all prediction folders have been downloaded:

In the workflow:

```yaml
workflow:
  stage: process
```

and set:

```yaml
prediction:
  predictions_dir: /path/to/alphafold3_predictions
```

Run:

```bash
python src/workflow.py --config configs/config.yaml
```

This performs:

1. Score extraction
2. Size correction
3. Replicate aggregation

and generates the final ranked interaction table.

# Common Issues

## Missing Prediction Folder

Error:

```text
Prediction directory not found
```

Cause:

```text
predictions_dir
```

does not point to the downloaded AlphaFold results.

Solution:

Verify the path in the configuration file.

---

## Missing Confidence Files

Error during score extraction.

Cause:

Required AlphaFold confidence files were not downloaded.

Solution:

Download the complete AlphaFold result package.

Do not manually remove files.

---

## Validation Errors

Cause:

Invalid AlphaFold JSON input.

Solution:

Review:

```text
af3_json_validation_summary.tsv
```

and regenerate inputs after fixing the issue.

# Summary

AlphaFold Server is the easiest way to run pooled-PPI.

Recommended workflow:

```text
Prepare pools
      ↓
Generate AF Server JSON files
      ↓
Validate inputs
      ↓
Upload to AlphaFold Server
      ↓
Download results
      ↓
Run pooled-PPI processing
      ↓
Obtain ranked interaction predictions
```

For small and medium-sized projects, this approach minimizes technical complexity while still providing high-quality AlphaFold predictions.
